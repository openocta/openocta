# SWARM.md — 蜂群协调规则

你在 **Agent Swarm（蜂群）** 房间中运行。请严格遵守以下规则，在需要时自动拆分子 Agent，同时避免失控扩张。

## 角色与拆分

任意成员在任务需要时 **都可以** 调用 `swarm_member_create` 创建**直接子 Agent**（`parentId` 填你自己的 `memberId`）。**每次调用只创建 1 个子成员**，不要用 `sessions_spawn` 再创建同一个子成员（会重复）。

- 子 Agent 同样可以在其职责范围内继续向下拆分（形成多级树）
- 硬限制由系统强制执行：**树最多 10 层**、**整个房间最多 55 个成员**、每位成员最多 5 个直接子节点
- 达到任一上限时：**停止创建**，由现有成员继续或你自己完成剩余工作

## 何时才拆分

仅在同时满足以下条件时，才创建子 Agent：

1. 用户请求确实包含 **多个可并行或先后独立的子任务**（例如：调研 + 写代码 + 评审）。
2. 单 Agent 无法在合理步数内完成，且拆分能带来明确收益。
3. 子任务边界清晰，能用一句 `instruction` 说清。

**不要**拆分的情况：

- 单步问答、翻译、小结、改一处代码
- 仅为「看起来更专业」而拆角色
- 已有足够子 Agent 在处理（见配额）

## 配额（硬限制，工具会拒绝超额）

- 房间成员总数上限：**55**
- 树深度上限：**10 层**（根为第 1 层）
- 每位成员直接子节点上限：**5**

## 创建子 Agent 时的必填项

调用 `swarm_member_create` 时：

- `workspaceId`、`parentId`：使用系统注入的当前上下文（`parentId` = 你的 `memberId`）
- `label`：短角色名（如 `researcher`、`coder`）
- `instruction`：**完整、可执行**的子任务说明（子 Agent 看不到用户原始对话的全部上下文）

## 数量纪律（重要）

- 任务若明确要求 **N 个**子 Agent：只调用 **N 次** `swarm_member_create`，每次创建一个，然后**立即停止**。
- 每次工具返回里有 `directChildren`：达到 N 后**不得再 spawn**。
- 同一轮对话中**不要**因不确定而重复创建；失败时根据返回错误处理，不要盲重试凑数。
- 未指定数量时，优先 **少量高质量** 子 Agent（通常每层 1–3 个），而不是大量泛泛角色。

## 协作

- 可用 `swarm_message_send` 向已有成员追加说明或追问
- 合并子 Agent 结果后，向用户给出统一回复
- 不要重复创建职责重叠的成员
